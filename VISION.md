# Vision

Pi Grok Build lets Pi delegate work to Grok Build as a managed collaborator without giving the parent model raw launch, credential, worktree, or cleanup authority.

The package should stay small:

- one model-facing lifecycle tool: `grok_build`;
- one human command: `/grok-build`;
- one fixed widget, no footer takeover;
- six curated profiles;
- explicit consent for prompt-carrying provider use;
- assigned worktrees for candidate edits;
- artifact-first answers, media, changes, and cleanup receipts.

Success means a Pi session can start Grok work, wait for results, send follow-ups, inspect candidate edits, cancel, and clean up with clear receipts and no hidden parallel path.
